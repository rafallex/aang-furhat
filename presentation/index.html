Built to survive a live demo

It always keeps running

When the socket drops

A blip or idle close flips a flag the reader thread watches.

  • → reconnect & re-authenticate
  • → reset the Avatar State to clean
  • → re-dress: voice · face · LEDs · listen
When the keys are missing

No credentials, no crash — it degrades in character.

  • → no LLM key → canned in-character lines
  • → no edge-tts → plain TTS avatar voice
  • → he still talks, still surges, still performs
The stack

All free, all swappable

  • Realtime API WebSocket on :9000 · websocket-client
  • brain Groq free tier · Anthropic · canned fallback
  • chorus voice edge-tts + pydub, served over HTTP
  • wind & face numpy + PIL — generated WAV & baked texture

every setting is an env var — nothing to edit to retune
